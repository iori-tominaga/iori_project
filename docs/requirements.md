# 要件定義書

**プロジェクト名:** JSTQB AL-TM 学習支援アプリ  
**バージョン:** 1.0.0  
**作成日:** 2026-04-12  
**最終更新日:** 2026-04-12

---

## 1. 目的・背景

### 1.1 目的
JSTQB Advanced Level Test Manager（以下 AL-TM）の資格取得を目指すユーザーが、模擬問題を繰り返し解くことで効率的に学習できるWebアプリを提供する。

### 1.2 背景
- AL-TM の試験は専門性が高く、体系的な問題演習が合格への近道である。
- 既存の学習ツール（テス友など）に近い体験を、JSTQB AL-TM 専用にカスタマイズして提供したい。
- 問題・解説はシラバスおよびWeb上の有志問題を参考に独自に作成し、DBで管理・追加していく。

---

## 2. ユーザーと利用シーン

### 2.1 対象ユーザー
- JSTQB AL-TM の受験を目指すテストエンジニア・テストマネージャー
- ログインなし（単一ユーザー利用を想定）

### 2.2 主な利用シーン
| シーン | 説明 |
|--------|------|
| 分野別学習 | 苦手な分野を選んで集中的に問題を解く |
| 通し学習 | 全問題をランダムに解いて総合的に確認する |
| 弱点確認 | 過去に間違えた問題や正答率の低い分野を中心に復習する |
| 進捗確認 | 分野ごとの正答率・学習状況をダッシュボードで確認する |

---

## 3. 機能要件

### 3.1 問題出題機能
| ID | 要件 |
|----|------|
| F-01 | 分野（シラバス章）を選択して問題を出題できる |
| F-02 | 全分野からランダムに問題を出題できる |
| F-03 | 難易度（易・普通・難）でフィルタリングして出題できる |
| F-04 | 出題数（セッションあたりの問題数）を選択できる |
| F-05 | 未回答の問題のみを絞り込んで出題できる |
| F-06 | 過去に間違えた問題のみを絞り込んで出題できる |

### 3.2 回答・採点機能
| ID | 要件 |
|----|------|
| F-07 | 4択選択式で回答する（1問1答） |
| F-08 | 回答後に即時フィードバック（正解・不正解）を表示する |
| F-09 | 正解の選択肢をハイライト表示する |
| F-10 | 選択した誤答の選択肢をハイライト表示する |

### 3.3 解説表示機能
| ID | 要件 |
|----|------|
| F-11 | 回答後に解説文を表示する（正解・不正解どちらでも表示） |
| F-12 | 解説はシラバスの該当箇所に基づいた内容とする |

### 3.4 学習進捗管理機能
| ID | 要件 |
|----|------|
| F-13 | 全体の正答率を表示する |
| F-14 | 分野ごとの正答率を表示する |
| F-15 | 回答履歴（日時・問題・正誤）を保存・表示する |
| F-16 | 苦手分野（正答率が低い分野）を視覚的にわかりやすく表示する |

### 3.5 問題データ管理
| ID | 要件 |
|----|------|
| F-17 | 問題・選択肢・解説をデータベースで管理する |
| F-18 | 問題はシラバス分野（章）に紐付けて管理する |
| F-19 | 問題に難易度（1:易・2:普通・3:難）を設定できる |
| F-20 | 問題はシードデータとして初期投入し、随時追加できる構造とする |

---

## 4. 非機能要件

### 4.1 性能
| ID | 要件 |
|----|------|
| NF-01 | 問題の表示は1秒以内に応答する |
| NF-02 | 回答履歴の集計・表示は3秒以内に応答する |

### 4.2 可用性・環境
| ID | 要件 |
|----|------|
| NF-03 | ローカル環境での動作を前提とする |
| NF-04 | Windows 11 + Node.js 環境で動作すること |

### 4.3 セキュリティ
| ID | 要件 |
|----|------|
| NF-05 | ログイン機能は不要（シングルユーザー利用） |
| NF-06 | SQLインジェクション等の基本的なセキュリティ対策を実施する |

### 4.4 保守性
| ID | 要件 |
|----|------|
| NF-07 | 問題データを追加・編集しやすい構造（DBシード）とする |
| NF-08 | フロントエンドとバックエンドを分離した構成とする |

---

## 5. スコープ外（対象外）

- ユーザー認証・ログイン機能
- 問題の管理画面（GUI）※初期はシードデータによる投入
- 複数ユーザー対応
- スマートフォン専用UI（レスポンシブ対応はするが、専用UIは作らない）
- 他資格（AL-TA、Foundation Level など）への対応（将来拡張として検討可）
- 本番環境へのデプロイ（初期はローカル運用）

---

## 6. 分野定義（シラバス章）

JSTQB Advanced Level Test Manager シラバスに基づく以下の7分野を対象とする。

| 分野ID | 分野名 | 説明 |
|--------|--------|------|
| 1 | テストプロセス | テストの計画・監視・制御・分析・設計・実装・実行・評価・クロージャ |
| 2 | テスト管理 | テスト管理全般、リスクベーステスト、見積もり、メトリクス、アウトソーシング |
| 3 | レビュー | マネジメントレビュー、フォーマルレビュー、インスペクション、メトリクス |
| 4 | 欠陥管理 | 欠陥ライフサイクル、欠陥レポート、プロセス能力評価 |
| 5 | テストプロセスの改善 | TMMi、TPI Next、CTP、STEP |
| 6 | テストツールと自動化 | ツール選択、ツールライフサイクル、ROI、CI/CD |
| 7 | 人的スキル・チーム構成 | 個人スキル、チームダイナミクス、モチベーション、コミュニケーション |
